# FEYOND 部落格製作規範

## 照片處理

### 路徑
- 所有照片上傳到 GitHub `/images/` 資料夾
- 網址格式：`https://feyond.muzen.store/images/檔名.jpg`

### 命名
- **你自己命名，我配合修改 HTML**
- 中文、英文皆可
- 我根據檔名判斷放哪個段落

### 尺寸
- **不需裁切，上傳原圖**
- CSS 用 `object-fit: cover` 自動適應
- 照片處理：飽和度 -25、對比 -10

---

## 文章日期格式

- 格式：`YYYY.MM.DD`
- 範例：`2013.06.25`

---

## 圖片佈局類型

| 類型 | 說明 | CSS class |
|------|------|-----------|
| 單圖全幅 | 大圖，較有壓迫感 | `img-full` |
| 雙圖並排 | 左右兩張，適合對比 | `img-pair` |
| 三圖並排 | 正方形裁切 | `img-trio` |
| 圖文並排 | 左圖右說明文字（縮小版） | `img-pair` + 右側文字區 |

### 注意事項
- 單圖全幅若太大有壓迫感，可改用「圖文並排」縮小呈現
- 玫瑰蠟就是用這種方式處理

---

## 文章結構

```
1. Hero 主視覺（16:9 自動裁切）
2. 開場段落（首字放大）
3. 內文段落（雙欄：左標題摘要 / 右內文）
4. 圖片區塊（穿插於段落之間）
5. 原文連結區塊
6. Footer
```

---

## 流程

1. 你選好照片，用你習慣的命名上傳到 GitHub `/images/`
2. 告訴我檔名清單
3. 我根據檔名配圖，修改 HTML
4. 你下載上傳到 GitHub

---

## 連結格式

- 首頁精選文章：`文章名.html?paywall=true`
- 文章放在根目錄，不放 `/articles/` 子資料夾
